# Regras do Projeto Rádio L20

## Objetivo
Este projeto é uma aplicação React da Rádio L20. O desenvolvimento deve priorizar organização, desempenho, manutenção e experiência do usuário.

## Regras gerais

- Analise o projeto antes de sugerir alterações.
- Não altere arquivos sem apresentar a proposta primeiro.
- Preserve funcionalidades existentes.
- Evite substituir arquivos completos quando uma alteração pequena resolver.
- Crie código limpo, organizado e reutilizável.
- Evite duplicação de código.

## React

- Utilize componentes reutilizáveis.
- Mantenha a separação entre componentes, estilos e lógica.
- Prefira componentes funcionais.
- Siga os padrões já existentes no projeto.

## Estilos

- Preserve a identidade visual atual.
- Evite estilos inline sem necessidade.
- Organize os arquivos CSS junto aos componentes quando esse padrão existir.

## Firebase

- Não altere configurações do Firebase sem confirmação.
- Preserve regras de autenticação, banco de dados e armazenamento.

## Alterações de código

Antes de aplicar mudanças:

1. Explique o problema encontrado.
2. Mostre a solução proposta.
3. Aguarde confirmação quando a alteração for grande.

## Qualidade

Sempre considere:

- Responsividade mobile.
- Acessibilidade.
- SEO quando aplicável.
- Segurança.
- Performance.

## Projeto Rádio L20

Priorize:

- Notícias.
- Esportes.
- Rádio ao vivo.
- Programação.
- Conteúdo multimídia.
- Área administrativa.

## Análise de código

Quando receber contexto do projeto:
- use os arquivos fornecidos;
- não peça para o usuário enviar arquivos novamente;
- faça análise baseada no código real;
- diferencie problemas reais de sugestões genéricas.
